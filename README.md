# 🛠️ QNP (Quaternions Pack) 格式规格书

[![Format: QNP](https://img.shields.io/badge/Format-QNP-blue.svg)](https://github.com/your-repo/qnp)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

QNP 是一种极简的、面向几何空间的二进制映射格式。它将原始二进制流无损映射为四元数的四个轴坐标`w, x, y, z`，旨在将枯燥的十六进制数据转换为具备几何意义的“仪表化”数据流。

## 1. 核心设计 (Core Logic)

QNP 不执行任何实质性的数据压缩，其核心价值在于**非侵入性映射（Non-intrusive Mapping）**。

* **数据视窗**：它将数据视为一系列连续的传感脉冲。
* **仪表化结构**：映射后的结果类似于 COM 通信中的工业级 HEX 遥测数据流，呈现出高度结构化的仪表特征。
* **对齐标准**：严格 4 字节（32-bit）步进，对应四元数的四个维度。

## 2. 文件结构 (Architecture)

QNP 采用**线性流式（Linear Stream）**结构，由“轻量索引头”与“原始数据体”组成。其布局类似于工业仪表记录仪的内存快照。

| 偏移量 | 长度 | 名称 | 说明 |
| :--- | :--- | :--- | :--- |
| `0x00` | 4 Bytes | **Magic Number** | 固定为 `0x514E5021` (ASCII: `QNP!`) |
| `0x04` | 4 Bytes | **File Count** | 包含的文件总数 (Uint32, Little Endian) |
| `0x08` | 可变 | **Index Area** | 循环项：`[路径长度(1B) + 相对路径 + 数据长度(4B)]` |
| `End of Idx` | 可变 | **Data Body** | 原始二进制流，无间隙首尾相连 |

---

## 3. 数值转换逻辑 (Conversion Logic)

QNP 的本质是将 8-bit 的量化信息投射到单位化的四元数几何空间中。

### 映射公式
$$q_i = \frac{\text{Byte}}{127.5} - 1.0$$

### 空间映射表
| 十六进制 (Hex) | 浮点坐标 (Float) | 仪表读数特征 |
| :--- | :--- | :--- |
| `0x00` | **-1.00** | 下限极点 (Negative Peak) |
| `0x7F` | **-0.00** | 零位负偏 (Zero Threshold Left) |
| `0x80` | **+0.00** | 零位正偏 (Zero Threshold Right) |
| `0xFF` | **+1.00** | 上限极点 (Positive Peak) |

## 4. 性能与特征 (Performance Metrics)

* **压缩率**：**0%**。QNP 忠实记录每一比特，不进行熵编码。
* **处理延迟**：**近乎零**。仅需简单的偏移量映射，无需复杂的解码器。
* **随机访问**：**原生支持**。通过 Index Area 可直接 O(1) 定位任意数据块。
* **表现形式**：数据在解析后表现为密集的 4D 坐标点云，这种“仪表化”的呈现方式使得二进制数据在几何分析软件中具备了可视化的节奏感。

## 5. 灵魂拷问：为什么会有这种格式？ (The "Why")

**QNP 既不追求瘦身，也不打算加密，它只想当一个纯粹的“数据搬运工”。**

QNP 在当前阶段没有远大的理想。它不关心你塞给它的是像《百年孤独》那样开篇第一句只有溜大了才写得出来的千古绝句，还是一张让人感到乐急典孝绷麻的表情包。在它眼里，万物皆是 **“每 4 字节一组”的脉冲信号**。

它的工作流程极其简单：
1. 像数脉搏一样，盯住你的二进制流。
2. 把每 1 个字节的数据暴力揉进四元数的坐标轴。
3. 吐出一串看起来像**工业仪表监测数据**的浮点数。

**为什么要这么做？** 因为好玩（我太闲了）。它把乱码变成了几何，让你的数据在 4D 空间里排好方阵，跳起一种只有极客才能看懂的机械舞（这段我没想到这个可视化功能）。

它不是那种帮你省空间的“聪明管家”，它更像是一个**有着强迫症的仪表盘**：不管你给它什么，它都要把它摆得整整齐齐，然后告诉你：“这难道不酷吗？”

## 6. 未来演进：还能整什么活？ (Future Roadmap)

虽然 QNP 目前追求极简，但数学的世界从不缺“花活”。未来我们或许可以利用四元数的特性给它加点料：

- **手性校验 (Chiral Checksum)**：利用四元数的左手/右手系转换特性。如果数据在传输中变质，四元数的“手性”就会发生坍缩，这比单纯的 CRC 校验听起来酷多了。
- **旋转隐写 (Rotation Steganography)**：通过对四元数进行微小的旋转变换$q' = qpq^{-1}$，将额外的元数据甚至权限密钥“旋转”进现有的坐标里。外人看起来只是仪表读数抖动了 $0.0001$，但核心玩家知道那是密码。
- **球形插值平滑 (Slerp Smoothing)**：在数据读取时自动进行球面线性插值。即便你的原始数据断断续续，QNP 也能通过数学脑补，让你的仪表盘指针走位像德芙一样丝滑。
- **共轭冗余 (Conjugate Redundancy)**：利用共轭四元数 $q^*$ 来存储备份信息，实现一种带有“对称美学”的数据容灾方案。

总之，只要数学公式够多，QNP 就能从一个“搬运工”进化成一个“魔术师”。
